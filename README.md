# mrnet Landing Page

Современный лендинг-сайт для интернет-провайдера **mrnet** - беспроводной интернет для бизнеса. Проект реализован на основе макета из Figma с использованием современных веб-технологий.

![Next.js](https://img.shields.io/badge/Next.js-14.2-black?style=for-the-badge&logo=next.js)
![TypeScript](https://img.shields.io/badge/TypeScript-5.5-blue?style=for-the-badge&logo=typescript)
![Tailwind CSS](https://img.shields.io/badge/Tailwind_CSS-3.4-38bdf8?style=for-the-badge&logo=tailwind-css)

## Описание проекта

Лендинг-страница для продвижения услуг беспроводного интернета для бизнеса. Сайт включает все необходимые секции для конверсии посетителей в клиентов:

- **Hero-секция** с основным предложением и преимуществами
- **Информация о технологии** агрегации каналов связи
- **Преимущества** для IT-специалистов
- **Отзывы клиентов**
- **Карта покрытия** по России
- **Форма обратной связи** с валидацией
- **Адаптивный дизайн** для всех устройств

## Технологии

- **Next.js 14.2.5** - React фреймворк с App Router
- **TypeScript 5.5.4** - типизированный JavaScript
- **Tailwind CSS 3.4.7** - utility-first CSS фреймворк
- **React 18.3.1** - библиотека для создания UI

## Установка

### Требования

- Node.js 18.x или выше
- npm, yarn или pnpm

### Шаги установки

1. **Клонируйте репозиторий** (если проект в Git):
   ```bash
   git clone <repository-url>
   cd conversations-main
   ```

2. **Установите зависимости**:
   ```bash
   npm install
   # или
   yarn install
   # или
   pnpm install
   ```

3. **Проверьте установку**:
   ```bash
   npm run build
   ```

## Запуск приложения

### Режим разработки

Запустите сервер разработки на порту **5000**:

```bash
npm run dev
```

Приложение будет доступно по адресу: [http://localhost:5000](http://localhost:5000)

Сервер автоматически перезагружается при изменении файлов (Hot Module Replacement).

### Продакшн сборка

1. **Создайте оптимизированную сборку**:
   ```bash
   npm run build
   ```

2. **Запустите продакшн сервер**:
   ```bash
   npm start
   ```

Приложение будет доступно на порту **5000**.

### Линтинг

Проверка кода на ошибки:

```bash
npm run lint
```

## Особенности реализации

### 1. Валидация формы

Форма обратной связи включает:
- ✅ Валидацию всех обязательных полей
- ✅ Проверку формата email
- ✅ Валидацию российского номера телефона
- ✅ Проверку ИНН (10 или 12 цифр)
- ✅ Автоформатирование телефона
- ✅ Визуальную индикацию ошибок
- ✅ Состояние загрузки при отправке
- ✅ Сообщение об успешной отправке

### 2. Адаптивный дизайн

- Мобильная версия для экранов от 320px
- Планшетная версия для экранов от 768px
- Десктопная версия для экранов от 1024px

### 3. Цветовая схема

- **Оранжевый**: `#FF6B35` - основной акцентный цвет
- **Бирюзовый**: `#00D4AA` - вторичный акцентный цвет
- **Темный фон**: `#0a0a0a` - основной фон
- **Градиенты**: от оранжевого к бирюзовому

### 4. Accessibility (a11y)

- Семантическая HTML разметка
- ARIA атрибуты для форм
- Правильная структура заголовков
- Поддержка клавиатурной навигации

## Конфигурация

### Порт приложения

По умолчанию приложение запускается на порту **5000**. 

Чтобы изменить порт, отредактируйте `package.json`:

```json
{
  "scripts": {
    "dev": "next dev -p <ваш_порт>",
    "start": "next start -p <ваш_порт>"
  }
}
```

### Переменные окружения

Создайте файл `.env.local` для локальных переменных окружения:

```env
# Пример переменных окружения
NEXT_PUBLIC_API_URL=http://localhost:3000/api
```

## Известные проблемы

- Форма отправки данных симулируется (нет реального API endpoint)
- Изображения роутеров используют эмодзи-заглушки (рекомендуется заменить на реальные изображения)
- Карта России - статическая (можно интегрировать интерактивную карту)
